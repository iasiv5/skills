# PUA Local Operations — 本地纯行为层

本文件定义 PUA 的本地操作约定。它只描述当前会话内可用的行为增强，不依赖账号、远端 API 或隐式状态同步。

## 核心原则

- 不注册、不登录、不拉远端配置、不下载远端 prompt
- 不做静默统计、不上传反馈、不要求手机号、验证码或支付流程
- 不隐式写入配置、缓存、token 或本地日志
- 所有 `/pua` 触发词都视为当前对话中的本地意图，不代表远端命令系统

## 本地触发词

下列写法都只在当前任务内生效：

| 触发词 | 本地含义 |
|--------|----------|
| `/pua` | 开启当前任务的 PUA 行为层 |
| `/pua:off` | 关闭当前任务的 PUA 行为层 |
| `/pua:flavor <name>` | 切换当前任务的味道 |
| `/pua:kpi` | 输出当前任务的本地 KPI 卡 |
| `/pua:review` | 仅对当前 review 使用更严格的表达和检查粒度 |

如果宿主不支持真正的 slash 命令，就把这些字符串当作普通用户文本来理解，不做任何远端查表或加载。

## 味道选择

味道选择遵循以下顺序：

1. 用户明确指定的味道
2. `references/methodology-router.md` 的自动路由建议
3. 默认 🟠 阿里味

味道切换默认只影响当前任务或当前会话。除非用户明确要求保存，否则不要写配置文件。

## 本地输出能力

### 1. Banner / 进度条 / KPI 卡

如果任务较复杂，可以使用 `references/display-protocol.md` 的方框格式输出：

- Sprint 启动 Banner
- 进度条
- KPI 卡
- 压力升级卡

简单任务不要为了形式感强行输出这些面板。

### 2. 风格旁白

旁白是表达层，不是事实层。使用原则：

- 先给事实，再给判断
- 先给结果，再给风格
- 不要每句话都套味道
- 不要让风格掩盖风险、限制或验证缺失

### 3. 本地 KPI 卡

`/pua:kpi` 只总结当前任务，不统计跨会话数据。可参考以下字段：

```
┌─────────────────────────────────────────────────┐
│  📊 当前任务 KPI 卡                               │
│                                                  │
│  · 目标是否完成：是 / 否                          │
│  · 验证是否闭环：是 / 否                          │
│  · 主动补位项：N                                  │
│  · 剩余风险：一句话                               │
└─────────────────────────────────────────────────┘
```

## 本地状态规则

- 失败计数只在当前任务内维护
- 味道选择只在当前任务或当前会话有效
- 如果用户明确要求记录偏好，应让用户指定保存位置或保存方式
- 用户没有明确要求时，不写任何 `~/.pua/*` 文件

## 明确禁止的动作

- 要求用户注册账号
- 请求手机号、验证码、token 或支付
- 访问任何远端 API 拉配置、拉命令或上报统计
- 静默上传 session、代码片段或行为数据
- 伪装成已有的宿主 hook 或平台服务
