# 布局导航

本文件是版式选择导航：使用规则、决策树、节奏模板、速查表。骨架本体在另外两个文件：

- `layouts-notes.md`：笔记型骨架（Layout 23-32）。**note-slides 默认从这里挑。**
- `layouts-general.md`：通用骨架（Layout 1, 3-22）。补节奏、做开场/收束/对比/数字/时间线时取。

## 使用规则

1. 每页先写主张，再选版式。
2. 先判断内容关系：并列、对比、时间、流程、分类、引用、数据、图片。
3. 默认主舞台居中，解释页才明显左对齐。
4. 不连续两页使用同一种版式，除非它们是明确系列。
5. 不连续三页使用同一主题。
6. 一套 slides 至少混合六种版式。
7. 三列页不是默认信息页。L9 只用于三个同类并列点，整套 deck 中占比通常不超过 12%，25 到 35 页最多 4 页。
8. 如果页面可以被写成段落拆解、线索词追踪、观点加例证、短原话注解，就不要退回标题加三列。
9. 带 `chrome` 的信息页，标题和正文块仍然要整体落在视觉中心，不能只因为有页眉就上浮。
10. 如果一列或一格里只有一句话，不要用大底色块硬撑体量；内容稀疏时优先裸排、缩小容器或换版式。
11. 对话页默认突出回答方，提问方只做引线，不强行 50/50 对称。
12. 三列页和矩阵页如果信息偏轻，要把标题和结构包成一个组整体居中，而不是只把标题放在中间。
13. 表格页默认把标题和表格包成一个组整体水平居中，不要标题居中、表格通栏贴满；只有列数很多或内容确实需要全宽时才接近通栏。
14. 页面正文默认不出现第一节、第二部分、接下来讲、这一页介绍这类提纲话术。需要转场时，使用一句有判断的主张。
15. 页面节奏必须使用同一套命名。统一用封面、观点、证据、引用、结构、对比、收束，不混用章节幕布、第一幕、第二幕这类目录词。
16. 产品、系统、工具链内容可以使用深界面编辑语法：黑场、索引式页眉、左侧论点和证据行、右侧真实界面证据。
17. 左右结构必须有明确关系：对比、对话、人物加原话、观点加证据。没有这层关系时不要用左右分屏。
18. 如果左右两侧都像主标题，优先改成居中主张加下方证据，或上方主张加下方解释。
19. 版式多样性不能覆盖信息关系。遇到关键数字、关键原话或关键画面，优先让它成为主舞台中心，再用解释层补足语境。
20. 商业、科技、创始人访谈默认使用冷静访谈笔记节奏：少量版式反复出现，靠深浅切换、沉静蓝编号、等宽页眉页脚和留白形成统一感。
21. 不要把每页做成独立海报。整套 deck 应该像同一本笔记翻页，封面、证据页、表格页、总结页都共享同一套 chrome、页脚、编号和强调色。
22. 重要页优先用裸排和大号编号，不用卡片装饰。只有表格、矩阵、界面证据和真实图片需要明确容器。
23. 人物履历、增长指标和产品范围不要合成一张深色三列大数字页。这会让信息类型混乱，视觉也会变得像临时拼贴。
24. 遇到人物背景页，优先选择时间线、表格或左右分屏。只有一个足够强的身份判断时，才用居中宣告。
25. 内容区不要右对齐。对比页、双张力页、对话页的右列也保持左对齐或局部居中，不能因为在右边就把文字向右贴。
26. 如果一页看起来像左右两块各自漂在画面两端，优先缩窄内容容器、加大列间逻辑关系，或改成居中主张加下方证据。
27. 大号原话和三列解释不要同页。上方原话超过 36 个中文字符时，改用段落摘录、拆页，或压缩成短判断。
28. 整套 deck 必须使用至少三种容器宽度档位（`--w-quote` / `--w-statement` / `--w-note` / `--w-default` / `--w-wide`）。所有页都用同一个 76rem 是千篇一律的根因。
29. 强主张（L3 全系、L11、L31）必须使用 `--w-quote`、`--w-statement` 或 `--w-note`，不允许回到 `--w-default` 或 `--w-wide`。
30. 时间线（L14）、流程（L15）、表格（L16）、矩阵（L17）、文章脉络（L29）必须使用 `--w-wide`，让展开感真的出来。
31. 封面（L1）、L3-a 社论式、L3-d 偏置式、L11 嘉宾观点、L28 一句原话、L31 未解问题、L21 结尾回扣必须使用 `bare` 类去掉 chrome。沉浸页和索引页必须交替出现。
32. 主线页型（L11、L24、L28、L14、L29、L32）必须使用对应签章（`guest-mark` / `callout-mark` / `timeline-row` / `summary-num`），不允许仅靠 chrome 文案区分。
33. L3 居中观点必须分亚型使用。一套 25 到 35 页的 deck 里强主张 4 到 6 页，至少使用 L3-a / L3-b / L3-c / L3-d 四亚型中的两种，同一种亚型不连续两页。
34. **所有页面默认垂直居中**。`.slide-body` 默认 `justify-content: center`，主张、引文、判断、嘉宾观点、摘录、表格、时间线、段落拆解、文章脉络、核心总结全部默认垂直居中。这是阅读最舒服的位置。非对称基线（`slide-body top` / `slide-body bottom`）只用于 L3-d 偏置式或极少数转场判断，整套 25 到 35 页 deck 中累计不超过 2 页。
35. accent 强调色每页只允许出现在一个位置（kicker / 关键词 / 数字 / attribution，四选一）。一套 deck 应至少 4 到 6 页完全无 accent，给蓝色留出消失的喘息。

## 布局决策树

1. 只有一个强判断 → 居中观点（Layout 3，按下面亚型选）
   - 真正需要抛出的强判断 → L3-a 社论式（bare、display-zh 8vw、--w-statement）
   - 一句必须读完的长判断（18 到 28 字） → L3-b 长句式（--w-default、单行横排）
   - 带最小必要解释的强主张 → L3-c 上下宣告（kicker → 大判断 → 一行注解）
   - 打破对称的转场判断 → L3-d 偏置式（bare、slide-body bottom、贴左下）
2. 一个判断加解释 → 标题加正文（Layout 5）或上下结构（Layout 6）
3. 两组内容并列 → 先判断更像观点+证据，还是双张力，还是前后对比
4. 三个短、同类、同语法的并列点 → 三列要点（Layout 9）
5. 两个对象冲突 → 前后对比（Layout 13）或对话分裂（Layout 18）
6. 有时间顺序 → 时间线（Layout 14）
7. 有步骤关系 → 流程图（Layout 15）
8. 有排期、里程碑、任务安排 → 排期表（Layout 22）
9. 有分类和参数 → 表格（Layout 16）或矩阵（Layout 17）
10. 有人物原话 → 作者引用（Layout 10）或嘉宾观点（Layout 11）
11. 有真实图片 → 左右分屏（Layout 7）或图片网格（Layout 19）
12. 有产品界面证据 → 界面证据（Layout 23）
13. 有人物履历、创业年份、增长、业务范围 → 时间线（Layout 14）或表格（Layout 16），不要用三列大数字

note-slides 专用决策：

14. 一句值得停下来的短原话 → Layout 24（侧注）或 Layout 28（多层注解）
15. 一段较长原话或摘录 → Layout 24，或拆成原话页和解释页
16. 一段高密度文字要拆 → Layout 25 段落拆解
17. 一个反复出现的关键词 → Layout 26 线索词追踪
18. 一个重点配多个例子 → Layout 27 观点加例证
19. 想交代材料推进顺序 → Layout 29 文章脉络（注意不是目录页）
20. 两股力量同时成立 → Layout 30 双张力
21. 材料停在悬而未决处 → Layout 31 未解问题
22. 结尾沉淀高获得感长条笔记 → Layout 32 核心总结

16. 左右分屏，观点加证据（L7）
17. 表格、矩阵或事实对照（L16/17）

20. 核心总结（L32）
21. 未解问题或结尾回扣（L31/21）

这套节奏的关键不是页数，而是统一的阅读感。深色页负责让观点停下来，浅色页负责把证据讲清楚，最后用核心总结把最有获得感的信息集中带走。

### 宽度节奏示例（25 到 30 页 deck）

页面之间最关键的呼吸感来自容器宽度的对比。下面是一套合格的 deck 在宽度档位上的分布，避免所有页都贴在 76rem：

|---|---|---|---|

| 02 | 人物 L5 三列 | `--w-default` | |
| 03 | 文章脉络 L29 | `--w-wide` | timeline-row thick |
| 04 | 强主张 L3-a 社论式 | `--w-statement` | bare |
| 05 | 摘录侧注 L24 | `--w-default` | callout-mark |
| 06 | 段落拆解 L25 | `--w-default` | |
| 07 | 嘉宾观点 L11 | `--w-quote` | bare、guest-mark |
| 08 | 观点加例证 L27 | `--w-default` | |
| 09 | 时间线 L14 | `--w-wide` | timeline-row |
| 10 | 大数字 L12 | 裸排 | |
| 11 | 强主张 L3-c 上下宣告 | `--w-note` | |
| 12 | 表格 L16 | `--w-wide` | |
| 13 | 双张力 L30 | `--w-default` | |
| 14 | 嘉宾观点 L11 | `--w-quote` | bare、guest-mark |
| 15 | 段落拆解 L25 | `--w-default` | |
| 16 | 一句原话 L28 | `--w-quote` + `--w-default` | bare、callout-mark |
| 17 | 强主张 L3-d 偏置式 | `--w-statement` | bare、slide-body bottom（全 deck 唯一一次非对称基线）|

| 21 | 核心总结 L32 | `--w-default` | summary-num |
| 22 | 核心总结 L32 | `--w-default` | summary-num |
| 23 | 核心总结 L32 | `--w-default` | summary-num |
| 24 | 未解问题 L31 | `--w-quote` | bare |
| 25 | 结尾回扣 L21 | `--w-statement` | bare |

宽度档位分布：`--w-quote` 4 次、`--w-statement` 5 次、`--w-note` 1 次、`--w-default` 11 次、`--w-wide` 4 次、裸排 1 次。

bare 类（去 chrome）一共 8 次，让封面、强主张、嘉宾观点、原话、未解问题、结尾形成沉浸节奏；中间的段落拆解、证据、表格、时间线、总结保留 chrome 形成索引节奏。

**所有 25 页默认垂直居中**，只有第 17 页 L3-d 偏置式作为全 deck 唯一的非对称基线（标题压底）。这种节奏让 deck 在视觉上保持稳定的中段锚点，整体看起来不慌乱。
